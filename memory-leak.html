Tentang Memory Leak

Memory Leak adalah kondisi di mana program mengalokasikan memori tapi tidak pernah membebaskannya (deallocate), sehingga memori yang terpakai terus bertambah sampai sistem kehabisan memori.

Penyebab Memory Leak:

  • Lupa memanggil free() atau delete
  • Circular references di struktur data
  • Event listeners tidak di-remove
  • Global variables yang terus bertambah
  • Cache yang tidak pernah dibersihkan

Dampak:

  • Performance degradation (sistem lambat)
  • Out of Memory (OOM) errors
  • System crash
  • Disk thrashing (swap berlebihan)
